3 правила для нормативных документов

1Вставляй текст напрямую в промт — не давай название файла. ИИ видит только то, что внутри промта.
2Запрети домысливание: «Если чего-то нет — пиши [УТОЧНИТЬ], не дополняй из своих знаний».
3Проси цитировать конкретный пункт для каждого требования — сразу видно, где ИИ придумал.

Общие правила промта

→Назначать роль с контекстом предметной области
→Докручивать несколькими итерациями и уточняющими вопросами
→Запрашивать примеры, сохранять в нужном формате
→В конце запросить итоговый промт — переиспользовать
→Для глубокой проработки включать Deep Thinking
→Чем больше контекста о системах и процессах — тем качественнее результат

Итерационные промты-помощники

1
Общее требование:
«Перепиши по шаблону: КОГДА / СИСТЕМА БЕРЁТ / ПРИМЕНЯЕТ ПРАВИЛО / ЗАПИСЫВАЕТ»
2
Лишнее:
«Проверь каждую строку — есть основание? Нет → [УТОЧНИТЬ]»
3
Исключения:
«Найди оговорки ("за исключением", "при условии"). Добавь в колонку»
4
Покрытие тестами:
«Какие правила из БТ не проверяет ни один тест-кейс? Добавь недостающие»

Типичные ошибки ИИ с нормативкой

✕Игнорирует перекрёстные ссылки между пунктами
✕Теряет «за исключением...» в конце пункта
✕Заменяет терминологию ЦБ «понятными» словами
✕Округляет или путает числовые пороги и диапазоны
✕Применяет устаревшую редакцию нормативного акта
